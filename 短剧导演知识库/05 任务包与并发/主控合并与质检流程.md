# 主控合并与质检流程

主控线程负责最终合并和质量控制。subagent 只做局部生成。

## 合并顺序

1. 检查任务包版本必须为 `TASK_SCHEMA_V4` 或更新；旧版本结果不得直接合并为最终稿。
2. 检查任务包是否标明并发成熟度和本集复杂度；缺失则不得合并。
3. 先验收 V4 证据包，再阅读正文。证据包缺项时不得由主控临时补证据，直接退回。
4. 按集数排序。
5. 按场次排序。
6. 按生成分镜单元排序。
7. 先剥离 V4 证据包，最终分镜正文不得混入证据表、导演表现层、小纲和过程分析。
8. 检查 V4 证据包是否完整：生成前六项证据卡完成状态、单元六项锁通过表、剧情覆盖简表、降尺度替代表达简表、专属动作词库简表、场景声场库简表、转场库简表、台词容量预判简表、高频词风险提示。
9. 检查 Seedance复杂度预算、镜头语言平衡结论和批量副作用风险提示是否完整；缺失时退回，不进入最终评分。
10. 检查时间轴是否连续。
11. 检查资产 ID 是否一致，且 subagent 没有自行新增共享资产。
12. 检查台词是否保留和归属正确。
13. 检查是否新增原剧本没有的剧情因果、线索、道具功能或人物行动结果；新增内容无原文依据时退回。
14. 检查导演表现层是否在执行层被执行。
15. 检查每个单元是否已在执行层完成8.5分导演小纲和单元六项锁。
16. 检查 C 类复杂集是否被过度拆分；连续情绪弧、多人群戏、闪回/现实交叉剪辑、集尾钩子被多方割裂时，整集退回。
17. 检查分镜格式是否统一。
18. 检查最终文档是否只保留可投喂分镜内容。
19. 检查 subagent 是否完成返回前自测；未提交自测结论或自测结论与正文明显冲突时，不进入软质量评分，先退回。

## V4证据包验收

主控合并时必须逐项验收：

- 生成前六项证据卡完成状态：六项均为已完成；任一项未完成，退回对应任务。
- 单元六项锁通过表：每个单元都明确通过；任一单元缺剧情结果、冲突点、功能锚点、动作链、反应升级或剪辑落点，退回重拆。
- 剧情覆盖简表：关键事件、关键台词信息、关系转折、伏笔/回收、离场状态或集尾方向均有承载。
- 降尺度替代表达简表：高风险信息只降尺度不删功能，不能改变人物动机和下一集钩子。
- 专属动作词库简表：场景动作、道具动作、空间动作能替代通用动作模板。
- 场景声场库简表：每个地点有识别度，不能跨地点共用兜底声效。
- 转场库简表：每个单元都有具体画面或声音落点，能在当前单元尾帧留下可剪的压力、情绪或动作余韵。
- 台词容量预判简表：长台词和同人连续台词已列出正文字数、计划秒数、目标字/秒、是否需要增单元；容量不够时已先重拆，不得靠压缩 1 秒反应或话落即切通过。
- Seedance复杂度预算结论：主动主体、道具、动作、场景和站位都在稳定生成范围内；元素过载、动作复杂或站位依赖前文时退回。
- 镜头语言平衡结论：中近景/近景为主，远景/全景和特写只服务明确功能；普通对话不得连续正面固定切；固定机位过密或清零都要复审。
- 批量副作用风险提示：孤儿字段、坏运镜词、提示词缩水/重复、停半拍滥用、宽泛主体回潮、正文媒体指令任一命中时，先退回上一可信基底或重做对应段落。
- 高频词风险提示：区分通用动作词和场景功能词；通用动作词高频且承担主要表演时退回。
- 返回前自测结论：字段、时长、声音、红线、手部词、重复度、剧情覆盖均已自测通过；任一项未测或失败，不合并。

## 质检入口

使用 [[分镜单元质检清单]]。

重点检查：
- 每个单元能否独立投喂 Seedance。
- 人物站位是否清楚。
- 台词时长是否足够。
- 是否有人物同时说话。
- 景别是否服务剧情。
- 动作是否有过程。
- 是否出现抽象情绪。
- 是否通过 8.5 分硬闸门；低于 8.5 的集数只能标记为结构草稿。
- 是否统计本集和本批次重复动作词、兜底声效、兜底转场。
- 是否完成 V4 证据包验收；证据包缺项的集数不能进入最终分镜。
- 是否完成 Seedance复杂度预算和镜头语言平衡检查；未完成时不得标记为 9 分样本。
- 是否完成真实并发证据验收；缺少任务包编号、临时结果、subagent 返回记录和主控合并记录时，只能记为主控分批生成，不能记为并发。

## 问题处理

- 小格式问题：主控线程统一修正。
- 资产缺失：标记待补资产。
- 剧情冲突：回到用户确认。
- 规则失效：更新 [[失败案例库]] 或 [[成功提示词模式库]]。
- V4 证据包缺项：退回生成前卡口重做。
- 台词容量预判缺失或明显错误：退回单元设计，先增单元或重拆边界。
- 单元六项锁失败：回到单元结构重拆，不做表层润色。
- 8.5 分硬闸门失败：回到单元导演小纲重拆，不做表层润色。
- 跨集模板污染：整批退回，先重做动作词库、声场和剪辑落点。
- Seedance元素过载：退回稳定性优化，减少主动主体、道具和复杂动作。
- 镜头语言失衡：退回镜头语言优化，修正远景长对白、连续特写、固定机位模板感或角度单一。
- 剧本外新增剧情污染：退回剧情覆盖表，删除或改成只服务原文的环境细节。
- 成熟度或复杂度标签缺失：不合并，退回主控补发任务包。
- C 类复杂集被拆碎导致主视角、动作链或集尾钩子断裂：整集退回，由单一执行者重写或主控亲自定稿。
