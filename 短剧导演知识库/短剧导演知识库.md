## 标准生产链路

[[剧本文本规范化规则]] -> [[剧本清洗闸门]] -> [[资产ID规则]] -> [[场次快照模板]] -> [[导演表现层规范]] -> 生成前六项证据卡 -> 8.5分导演小纲 -> 台词节奏预排表 -> 台词容量闸门 -> [[分镜单元写作规范]] -> [[分镜单元质检清单]]

硬规则：不得跳过 [[导演表现层规范]]、生成前六项证据卡、8.5分导演小纲、台词节奏预排表和台词容量闸门，直接从原文动作/对白批量生成最终分镜。没有在执行层完成场次快照、导演表现层、剧情覆盖表、降尺度替代表达表、专属动作词库、场景声场库、转场库、单元导演小纲、连续同人台词总时长检查、台词容量预判和质检复盘的输出，只能标记为结构草稿，不能标记为最终分镜脚本。

## 重点规则

- [[双层提示词结构]]：内部协作用 ID 和短码省 token，最终投喂 Seedance 时展开为完整分镜单元。
- [[跨项目分镜生产调用规则]]：其他项目复用的总执行口径，包含项目圣经、资产补充、源文锁定、正推反推、台词节奏、语义拆句、9分质检和双目录同步。
- [[多角色团队制分镜生产规则]]：主控自动组织制作、稳定性优化、镜头语言优化、质检审批和记忆更新，不把未过审半成品交给用户。
- [[9分画面导演闸门]]：区分8.5可用稿和9分批量样本，强制检查核心画面句、主视觉锚点、Seedance执行预判、多人主体控制和单元尾气口。
- [[剧本文本规范化规则]]：统一引号、冒号、空格和对白/OS/VO格式。
- [[时长与台词计算规则]]：防止台词说不完、分镜太赶或拖拉；同一人物连续台词必须按整段计算总时长，并按语义节点拆分。
- [[景别曲线规则]]：景别服务剧情强度，不为炫技。
- [[低AI感规则]]：用真实表演细节、动作过程、镜头控制降低 AI 感。
- [[打斗单元提示词规则]]：打斗单元使用独立动作补充提示；全剧完成后可筛选 A/B 级单元单独生成 9/12 格动作故事板提示词。
- [[最终规则返修经验链路]]：沉淀《下山后》返修暴露的源文锁定、标点节奏、画面公式、自动拆分占位词、差异化评分和双目录同步规则。
- [[全局复盘与踩坑日志]]：沉淀跨项目长期有效规则和踩坑清单，后续 agent 进入导演项目、批量生成、返修或审计前先读。
- [[只读快照规则]]：支持 subagent 并发，避免共享文档争抢。

## 使用建议

1. 拿到新剧本后，先走 [[剧本文本规范化规则]] 和 [[剧本清洗闸门]]。
2. 建立资产 ID，但当前阶段只登记，不负责生成图片。
3. 每场先写 [[场次快照模板]]，再写 [[导演表现层规范]]。
4. 生成分镜时严格使用 [[分镜单元写作规范]]。
5. 每次生成后用 [[分镜单元质检清单]] 检查。
6. 失败结果写入 [[失败案例库]]，成功模式写入 [[成功提示词模式库]]。

## 批量生产闸门

- 目标质量线为 8.5 分；低于 8.5 的集数不得标记为最终分镜脚本。
- 每次批量生成前，先完成 1-3 集 8.5 分样本并由主控复盘，样本通过后才能扩展批量。
- 批量范围建议 3-5 集一批；超过 5 集必须拆批，并在每批后做重复度、声场、转场、人物主体一致性质检。
- 程序化生成器只能用于结构草稿、时长粗分、字段整理；不得直接产出“最终分镜脚本”。
- 最终稿必须经过人工导演层改写：每场至少明确观众视角、主动作链、声场、剪辑落点和禁用表达。
- 7.5 分靠质检，8.5 分靠生成前导演设计。每个单元生成前必须在执行层先过“剧情结果、冲突点、功能道具/空间锚点、动作链、反应升级、镜头节奏、剪辑落点、风险规避”八项小纲；小纲默认不输出到最终分镜文档。
- 审核允许少量修正和退回重写，但基础质量问题要尽量在生成前解决。每集生成正文前必须先在执行层完成六项前置证据卡：剧情覆盖表、单元导演小纲、专属动作词库、场景声场库、转场库、降尺度替代表达表。缺任一项，不允许写最终分镜正文。
- 每个单元写正文前必须通过六项单元锁：剧情结果、冲突点、功能锚点、动作链、反应升级、剪辑落点。缺任一项，先重拆单元，不允许靠成稿后润色补救。
- 每集写正文前必须完成内部台词节奏预排表：原文台词、说话人、语义节点、建议总时长、节点分镜时长、是否需要听者反应/道具变化/空间压力。没有预排表，不允许写最终分镜正文。
- 每集写正文前必须完成台词容量闸门：源文台词、必要反应和尾帧气口放不进 15 秒时，先增单元或重拆边界，不沿用旧稿单元数硬塞。
- 同一人物连续说话必须做整段总时长检查，不能只检查单条分镜语速。整段低于 2.6 字/秒、3 镜以上重复同一动作、或拆点没有语义依据，均退回语义拆分和导演小纲，不允许靠表面润色通过。
- 8.5可用稿不能自动当9分样本。每批样本必须通过 [[9分画面导演闸门]]：核心画面句、主视觉锚点、Seedance执行预判、多人主体控制、OS/VO不过度解释、单元尾气口全部成立。
- 低于 8.5 的单元不做局部润色，必须回到单元导演小纲重拆。
